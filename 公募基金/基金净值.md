# 基金净值（MCP 工具 `ft_get_fund_nav`）

> **MCP 工具**：`ft_get_fund_nav`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：按基金代码分页查询基金净值序列（单位净值/累计净值/复权净值/日增长率/每万份收益/7日年化/申赎状态/分红配送）。校验 `institution_code` 非空，默认 `page=1`、`page_size=50`。提示：`institution_code` 必填；时间序列数据，按 `end_date`（截止日期）排序分页。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，默认 `page=1`、`page_size=50`
- 提示：
  - `institution_code` 必填。
  - 时间序列数据，按 `end_date`（截止日期）排序分页。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| institution_code | string | Y | 基金代码 |
| page | int | N | 当前页，默认 1 |
| page_size | int | N | 每页条数，默认 50 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

FundNavResponse：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| security_code | string | Y | 基金代码 |
| security_name | string | Y | 基金名称 |
| end_date | int | Y | 截止日期 YYYYMMDD |
| nv | decimal | Y | 资产净值 |
| unitnv | decimal | Y | 单位净值 |
| unitnv_grw | decimal | Y | 单位净值日增长率（%） |
| accum_unitnv | decimal | Y | 累计净值 |
| prof_d | decimal | Y | 每万份基金收益（元） |
| annurtn_7d | decimal | Y | 7日年化收益率 |
| subscribe_status | string | Y | 申购状态 |
| redeem_status | string | Y | 赎回状态 |
| dividend | string | Y | 分红配送 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_nav`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_nav","arguments":{"institution_code": "110011"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_nav', {"institution_code": "110011"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
