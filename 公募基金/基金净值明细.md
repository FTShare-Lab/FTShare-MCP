# 基金净值明细（MCP 工具 `ft_get_fund_net_value`）

> **MCP 工具**：`ft_get_fund_net_value`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：按 `fund_code` 查询基金净值明细。`nav_date` 单日或 `start_date`/`end_date` 区间（互斥）。含发布/净值日期、资产净值、单位净值、单位净值增长率、累计净值、复权因子、复权单位净值、复权单位净值增长率、每万份基金收益、7 日年化等 14 个字段。提示：`fund_code` 必填；`nav_date` 与 `start_date`+`end_date` 。实时参数约束以 `tools/list` 返回的 `inputSchema` 为准。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，默认 `page=1`、`page_size=50`（部分工具上限 200）

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| end_date | integer | N | 净值结束日期 YYYYMMDD |
| fund_code | string | Y | 基金代码 |
| nav_date | integer | N | 净值日期 YYYYMMDD（与 start/end 互斥） |
| page | integer | N | 页码，默认 1 |
| page_size | integer | N | 每页条数，默认 50，上限 200 |
| start_date | integer | N | 净值开始日期 YYYYMMDD（需与 end_date 同传） |

未在 `inputSchema` 中声明的字段会被拒绝。

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| fund_code | string | Y | 基金代码 |
| fund_name | string | Y | 基金名称 |
| trade_code | string | Y | 交易代码 |
| publish_date | int | Y | 发布日期 YYYYMMDD |
| nav_date | int | Y | 净值日期 YYYYMMDD |
| net_asset | string | Y | 资产净值 |
| unit_nav | string | Y | 单位净值 |
| unit_nav_growth | string | Y | 单位净值增长率（%） |
| accumulated_nav | string | Y | 累计净值 |
| adjustment_factor | string | Y | 复权因子 |
| adjusted_unit_nav | string | Y | 复权单位净值 |
| adjusted_unit_nav_growth | string | Y | 复权单位净值增长率（%） |
| daily_profit | string | Y | 每万份基金收益 |
| seven_day_annualized_return | string | Y | 7 日年化收益率 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_net_value`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_net_value","arguments":{"fund_code": "000001", "start_date": 20260101, "end_date": 20260717, "page": 1, "page_size": 2}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_net_value', {"fund_code": "000001", "start_date": 20260101, "end_date": 20260717, "page": 1, "page_size": 2})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
