# 基金列表（MCP 工具 `ft_get_fund_list`）

> **MCP 工具**：`ft_get_fund_list`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：分页查询公募基金基础列表。可按 `fund_code` 查单只、按 `fund_type` 过滤。含基金代码、名称、交易代码、基金公司、经理、托管人、运作方式、类型、成立日期、成立规模、上市日期、存续起止、业绩基准、最低申购/认购金额等 16 个字段。提示：`fund_type` 为自由字符串精确匹配（如 `股票型`/`混合型`/`债券型`/`货币型`/`保本型`/`其他型`/`REITs`），无。实时参数约束以 `tools/list` 返回的 `inputSchema` 为准。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，默认 `page=1`、`page_size=50`（部分工具上限 200）

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| fund_code | string | N | 基金代码 |
| fund_type | string | N | 基金类型，精确匹配（股票型/混合型/债券型/货币型/保本型/其他型/REITs） |
| page | integer | N | 页码，默认 1 |
| page_size | integer | N | 每页条数，默认 50，上限 200 |

未在 `inputSchema` 中声明的字段会被拒绝。

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| fund_code | string | Y | 基金代码 |
| fund_name | string | Y | 基金名称 |
| trade_code | string | Y | 交易代码 |
| fund_company | string | Y | 基金公司 |
| fund_manager | string | Y | 基金经理 |
| custodian | string | Y | 托管人 |
| operation_mode | string | Y | 运作方式 |
| fund_type | string | Y | 基金类型 |
| established_date | int | Y | 成立日期 YYYYMMDD |
| established_size | string | Y | 成立规模（元） |
| listed_date | int | Y | 上市日期 YYYYMMDD（0=未上市） |
| start_date | int | Y | 存续期起始日 |
| end_date | int | Y | 存续期截止日 |
| benchmark | string | Y | 业绩比较基准 |
| min_purchase_amount | string | Y | 最低申购金额 |
| min_subscription_amount | string | Y | 最低认购金额 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_list`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_list","arguments":{"page": 1, "page_size": 2}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_list', {"page": 1, "page_size": 2})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
