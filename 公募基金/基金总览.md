# 基金总览（MCP 工具 `ft_get_fund_overview`）

> **MCP 工具**：`ft_get_fund_overview`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：分页查询所有基金概览（近两个交易日单位/累计净值、日增长值与增长率、基金类别、申购状态）。
- 数据范围：最新快照（概览为最近两个交易日净值聚合，无历史时间序列）
- 单次限量：分页返回，由 `page` / `page_size` 控制
- 提示：
  - 无净值的基金对应字段为 `null`。
  - `daily_growth_ratio` 保留两位小数。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| page | int | Y | 当前页 |
| page_size | int | Y | 每页条数 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

FundOverviewItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| institution_code | string | Y | 基金代码 |
| institution_name | string | Y | 基金名称 |
| last_trade_day_unitnv | decimal | Y | 上一交易日单位净值（无数据 null） |
| last_trade_day_accum_unitnv | decimal | Y | 上一交易日累计净值（无数据 null） |
| second_last_trade_day_unitnv | decimal | Y | 上上交易日单位净值（无数据 null） |
| second_last_trade_day_accum_unitnv | decimal | Y | 上上交易日累计净值（无数据 null） |
| daily_growth_value | decimal | Y | 日增长值（无数据 null） |
| daily_growth_ratio | decimal | Y | 日增长率（%，两位小数，无数据 null） |
| institution_type | string | Y | 基金类别 |
| subscription_status | string | Y | 申购状态 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_overview`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_overview","arguments":{"page": 1, "page_size": 3}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_overview', {"page": 1, "page_size": 3})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
