# 基金持仓明细（MCP 工具 `ft_get_fund_portfolio`）

> **MCP 工具**：`ft_get_fund_portfolio`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：按 `fund_code` 查基金报告期持仓明细。`report_date` 单期或 `start_date`/`end_date` 区间（互斥），`publish_date` 按发布日期过滤。未传日期默认返回最新一期。提示：`fund_code` 必填；`report_date` 与 `start_date`+`end_date` 互斥；未传日期默认最新一期。实时参数约束以 `tools/list` 返回的 `inputSchema` 为准。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，默认 `page=1`、`page_size=50`（部分工具上限 200）

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| end_date | integer | N | 报告期结束日期 |
| fund_code | string | Y | 基金代码 |
| page | integer | N | 页码，默认 1 |
| page_size | integer | N | 每页条数，默认 50，上限 200 |
| publish_date | integer | N | 发布日期 YYYYMMDD |
| report_date | integer | N | 报告期 YYYYMMDD（与 start/end 互斥） |
| start_date | integer | N | 报告期起始日期（需与 end_date 同传） |

未在 `inputSchema` 中声明的字段会被拒绝。

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| fund_code | string | Y | 基金代码 |
| fund_name | string | Y | 基金名称 |
| trade_code | string | Y | 交易代码（部分债券/转债可能为空） |
| publish_date | int | Y | 发布日期 YYYYMMDD |
| report_date | int | Y | 报告期 YYYYMMDD |
| report_type | string | Y | 报告类型 |
| asset_type | string | Y | 资产类型 |
| asset_type_group | string | Y | 资产类型分组 |
| holding_code | string | Y | 持仓代码 |
| holding_name | string | Y | 持仓名称 |
| holding_order | int | Y | 持仓序号 |
| holding_shares | string | Y | 持仓数量 |
| market_value | string | Y | 持仓市值 |
| net_value_ratio | string | Y | 占基金净值比例（%） |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_portfolio`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_portfolio","arguments":{"fund_code": "000001", "report_date": 20251231, "page": 1, "page_size": 2}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_portfolio', {"fund_code": "000001", "report_date": 20251231, "page": 1, "page_size": 2})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
