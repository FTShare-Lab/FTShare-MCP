# 指数跟踪基金（MCP 工具 `ft_get_fund_index_fund`）

> **MCP 工具**：`ft_get_fund_index_fund`（category: `公募基金`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：按指数代码查询跟踪该指数的基金，默认返回全市场（场内 ETF + 场外联接基金），`scope=etf` 仅返回场内 ETF。`index_code` 支持裸码或带后缀。直接返回数组（无分页包装），按 `fund_code` 升序；无匹配返回空数组。提示：`index_code` 必填，支持裸码（如 `000300`）或带后缀（如 `000300.SH`、`H30198.CSI`）；`scope`。实时参数约束以 `tools/list` 返回的 `inputSchema` 为准。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，默认 `page=1`、`page_size=50`（部分工具上限 200）

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| index_code | string | Y | 指数代码，支持裸码或带后缀 |
| scope | string | N | `all` 全市场（默认）/ `etf` 仅场内 ETF |

未在 `inputSchema` 中声明的字段会被拒绝。

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| fund_code | string | Y | 基金代码（带 .OF/.SH/.SZ 后缀） |
| name | string | Y | 基金简称 |
| fullname | string | Y | 基金全称 |
| index_code | string | Y | 跟踪指数代码 |
| index_name | string | Y | 跟踪指数名称 |
| index_type | string | Y | 指数类型 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_fund_index_fund`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_fund_index_fund","arguments":{"index_code": "000300", "scope": "all"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_fund_index_fund', {"index_code": "000300", "scope": "all"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
