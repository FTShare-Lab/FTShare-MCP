# 新闻语义搜索（MCP 工具 `ft_semantic_search_news_handler`）

> **MCP 工具**：`ft_semantic_search_news_handler`（category: `大模型语料`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：按语义查询相关新闻，支持按年份或时间范围缩小搜索范围。提示：`query` 必填；`year` 与 `start_time`/`end_time` 任选一种时间条件。
- 数据范围：依赖外部语义检索服务的数据范围（实时检索，无固定时间维度）
- 单次限量：返回条数由 `limit` 控制，默认由服务端决定
- 提示：
  - `query` 必填。
  - 同时传 `start_time` / `end_time` 时，结束时间不能早于开始时间。
  - 接口转调外部服务，开放路由，可用性与延迟可能受外部服务影响。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| query | string | Y | 搜索文字 |
| limit | int | N | 返回条数，默认由服务端决定 |
| year | int | N | 年份，限定搜索范围 |
| start_time | datetime | N | 起始时间（带时区） |
| end_time | datetime | N | 结束时间（带时区） |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

SearchResult：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| news_id | uint64 | Y | 新闻 ID |
| source_site | string | Y | 来源站点 |
| article_url | string | N | 文章链接 |
| publish_time | datetime | N | 发布时间 |
| fetch_time | datetime | Y | 抓取时间 |
| title | string | N | 标题 |
| media_name | string | N | 媒体名称 |
| summary | string | N | 摘要 |
| content | string | Y | 正文内容 |
| is_truncated | int8 | Y | 是否被截断（0/1） |
| is_reviewed | int8 | Y | 是否已审核（0/1） |
| score | float64 | Y | 搜索匹配分数 |

## 调用方法（MCP）

> MCP 工具名 `ft_semantic_search_news_handler`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_semantic_search_news_handler","arguments":{"query": "宁德时代"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_semantic_search_news_handler', {"query": "宁德时代"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
