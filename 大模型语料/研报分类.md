# 研报分类（MCP 工具 `ft_type_reports`）

> **MCP 工具**：`ft_type_reports`（category: `大模型语料`）。返回 **markdown 表格文本**（非结构化 JSON）。输入参数 / 输出参数 / 数据样例对照 [ftshare-doc](../../ftshare-doc/api-doc) 原接口。

- 描述：按研报类型 + 日期范围查询研报，直接查 ClickHouse（不经 Redis 缓存）。支持 5 种研报类型：`MacroReport`（宏观）/ `IndustryReport`（行业）/ `BrokerMorningReport`（券商晨报）/ `StrategyReport`（策略）/ `NewStockReport`（新股）。返回与「研报列表」相同的 `StockReportItem` 结构（`PaginatedResponse`）。日期范围校验不能超过 3 天。
- 数据范围：以服务端返回为准
- 单次限量：分页返回，`page` / `page_size` 必填；`start_date` / `end_date` 区间跨度不能超过 3 天
- 提示：
  - `rept_type` 必填，取值仅限 5 类：`MacroReport` / `IndustryReport` / `BrokerMorningReport` / `StrategyReport` / `NewStockReport`。
  - `start_date` 必填；`end_date` 不填时默认等于 `start_date`（查单日）。
  - `start_date` / `end_date` 格式 `YYYYMMDD`，区间不能超过 3 天。
  - 出参结构同「研报列表」`StockReportItem`，`rating_change` 为中文。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| rept_type | string | Y | 研报类型：MacroReport / IndustryReport / BrokerMorningReport / StrategyReport / NewStockReport |
| start_date | string | Y | 开始日期 YYYYMMDD |
| end_date | string | N | 结束日期 YYYYMMDD，不填默认与 `start_date` 相同 |
| page | int | Y | 页码 |
| page_size | int | Y | 每页条数 |

## 输出参数

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| items | array[StockReportItem] | Y | 研报列表 |
| total_pages | int | Y | 总页数 |
| total_items | int | Y | 总记录数（未分页前） |

StockReportItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| stock_code | string | Y | 股票代码 |
| stock_name | string | Y | 股票名称 |
| report_id | string | Y | 研报 id |
| report_title | string | Y | 研报标题 |
| report_time | string | Y | 发布时间 YYYYMMDD |
| report_institution | string | Y | 研究机构（序列化名 `report_Institution`） |
| report_rating_name | string | Y | 本次评级名称 |
| report_rating_value | string | Y | 本次评级等级 |
| last_report_rating_name | string | Y | 上次评级名称 |
| last_report_rating_value | string | Y | 上次评级等级 |
| rating_change | string | Y | 评级变动中文：维持 / 上调 / 下调 / 首次覆盖 |
| rating_researcher | string | Y | 研究员 |
| url_hash | string | Y | 研报文件 URL 哈希 |

## 调用方法（MCP）

> MCP 工具名 `ft_type_reports`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，再 `tools/call`**，后续请求带该 header。返回 **markdown 表格文本**。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '')

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_type_reports","arguments":{"rept_type": "MacroReport", "start_date": "2026-06-20", "page": 1, "page_size": 3}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_type_reports', {"rept_type": "MacroReport", "start_date": "2026-06-20", "page": 1, "page_size": 3})
            print(res.content[0].text)   # markdown 表格文本

asyncio.run(main())
```
