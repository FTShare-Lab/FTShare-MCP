# 美国经济指标（MCP 工具 `ft_us_economic`）

> **MCP 工具**：`ft_us_economic`（category: `宏观经济/国际宏观`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询美国宏观经济指标时间序列。通过 `type` 指定指标类型，返回该指标的时间（年月或季度）、前值、现值、发布日期。统一接口覆盖 16 类指标：ISM 制造业/非制造业、非农就业、贸易帐、失业率、PPI 环比、CPI 环比/同比、核心 CPI 环比/同比、新屋开工、成屋销售、耐用品订单环比、咨商会消费者信心、GDP 同比初值、联邦基金利率上限。提示：`type` 必填，取值见输入参数表；`release_date` 为发布日期，ISO 格式 YYYY-MM-DD。
- 数据范围：美国宏观经济月度/季度数据，以服务端返回为准
- 单次限量：全量序列一次返回，无分页上限
- 提示：
  - `type` 必填，取值见输入参数表；非法值返回参数错误。
  - `month` 为展示用时间标签：月度指标为「YYYY年M月」，季度指标（如 GDP 同比初值）为「YYYY年第N季度」。
  - `release_date` 为发布日期，ISO 格式 YYYY-MM-DD。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| type | string | Y | 指标类型，枚举值见下表 |

`type` 枚举（kebab-case）：

| 取值 | 含义 |
|------|------|
| ism-manufacturing | ISM 制造业指数 |
| ism-non-manufacturing | ISM 非制造业指数 |
| nonfarm-payroll | 非农就业人数 |
| trade-balance | 贸易帐 |
| unemployment-rate | 失业率 |
| ppi-mom | PPI 环比 |
| cpi-mom | CPI 环比 |
| cpi-yoy | CPI 同比 |
| core-cpi-mom | 核心 CPI 环比 |
| core-cpi-yoy | 核心 CPI 同比 |
| housing-starts | 新屋开工 |
| existing-home-sales | 成屋销售 |
| durable-goods-orders-mom | 耐用品订单环比 |
| cb-consumer-confidence | 咨商会消费者信心 |
| gdp-yoy-preliminary | GDP 同比初值（季度） |
| fed-funds-rate-upper | 联邦基金利率上限 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

数组元素（UsIsmManufacturing，全 16 类共用）：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| month | string | Y | 时间标签，月度「YYYY年M月」/ 季度「YYYY年第N季度」 |
| prev_value | decimal | Y | 前值（上一期数值） |
| current_value | decimal | Y | 现值（当期数值） |
| release_date | string | Y | 发布日期，YYYY-MM-DD |

## 调用方法（MCP）

> MCP 工具名 `ft_us_economic`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_us_economic","arguments":{"type": "cpi-yoy"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_us_economic', {"type": "cpi-yoy"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
