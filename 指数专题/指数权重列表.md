# 指数权重列表（MCP 工具 `ft_index_weight_list_handler`）

> **MCP 工具**：`ft_index_weight_list_handler`（category: `指数专题/指数成分和权重`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：分页查询单只中证系列指数指定期次的成份股权重，返回成份代码、成份名称、日期和权重。提示：`index_code` 必填；`date` 可选，不传时查询最新可用期次。
- 数据范围：无时间维度，最新若干期快照（沪深300 当前可取到 2026-05-29）
- 单次限量：分页，`page_size` 默认 20、最大 100
- 提示：
  - 路径含 `index` 子前缀：`/market/data/index/index_weight`。
  - `index_code` **必填**，为空会被拒（400）。
  - `date` 可选；建议先从 `index_weight_summary` 拿到可用 `date` 再按期查。
  - `weight` 为字符串形式的小数权重（如 `0.0609`），非百分比。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| index_code | string | Y | 指数代码，如 000300（沪深300），为空会被拒 |
| date | string | N | 权重采集日期 YYYYMMDD（如 20260529），不传取最新期 |
| page | int | N | 页码，默认 1 |
| page_size | int | N | 每页条数，默认 20，最大 100 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| total | int | Y | 命中总条数（跨多期时为所有期成份累计） |
| page | int | Y | 当前页码 |
| page_size | int | Y | 当前每页条数 |
| index_weights | array | Y | 当页成份权重明细列表 |

index_weights 每项（IndexWeightInfo）：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| date | string | Y | 权重采集日期 YYYY-MM-DD |
| index_code | string | Y | 指数代码，如 000300 |
| component_code | string | Y | 成份股代码，如 601398 |
| component_name | string | Y | 成份股名称，如 工商银行 |
| weight | string | Y | 权重（小数字符串，如 0.0609） |
| url_hash | string | Y | 文件下载 hash，用于下载该期 xlsx 权重文件 |

## 调用方法（MCP）

> MCP 工具名 `ft_index_weight_list_handler`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_index_weight_list_handler","arguments":{"index_code": "000300"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_index_weight_list_handler', {"index_code": "000300"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
