# 申万行业成份股历史（MCP 工具 `ft_sw_industry_constituent_history`）

> **MCP 工具**：`ft_sw_industry_constituent_history`（category: `指数专题/申万行业`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询指定申万行业的成份股调入调出历史，返回股票代码、行业层级、调入日期和调出日期。提示：`industry_code` 必填，需带 `.SI` 后缀，例如 `801010.SI`。
- 数据范围：成份股调入日期可追溯至 2012 年（如 002688 农林牧渔成份 2012-01-05 调入），反映申万分类历史变更
- 单次限量：无分页，一次返回该行业全部成份股历史
- 提示：
  - `industry_code` 必填，带 `.SI` 后缀，如 801010.SI。
  - 返回非分页，`data.items` 为全部记录，`data.total` 为数量。
  - `inDate`/`outDate` 可为 null（仍在职）。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| industry_code | string | Y | 行业代码，带 .SI 后缀，如 801010.SI |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

SwIndustryConstituentHistoryItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| stockCode | string | Y | 股票代码（6 位） |
| stockName | string | Y | 股票名称 |
| inDate | string | N | 调入日期 |
| outDate | string | N | 调出日期（在职为 null） |
| swLevel1Code | string | Y | 一级行业代码 |
| swLevel1Name | string | Y | 一级行业名称 |
| swLevel2Code | string | N | 二级行业代码 |
| swLevel2Name | string | N | 二级行业名称 |
| swLevel3Code | string | N | 三级行业代码 |
| swLevel3Name | string | N | 三级行业名称 |

## 调用方法（MCP）

> MCP 工具名 `ft_sw_industry_constituent_history`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_sw_industry_constituent_history","arguments":{"industry_code": "801010.SI"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_sw_industry_constituent_history', {"industry_code": "801010.SI"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
