# 申万行业日度指标（MCP 工具 `ft_sw_industry_daily_metrics`）

> **MCP 工具**：`ft_sw_industry_daily_metrics`（category: `指数专题/申万行业`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询单个申万行业的日频估值/资金/量价指标时间序列，含 PE（LYR/TTM 及分位/中位数）、PB、股息率、换手率、总市值、收盘价、破净比例、指数收盘、成交额及拥挤度等约 40 项字段（字段全部为字符串）。按行业层级（1/2/3）分表查询，单行业约 1200+ 个交易日。提示：`level` 必填（1/2/3），`start_date`/`end_date`/`industry_code` 必填；日期格式 YYYYMMDD；`industry_code` 必须带 `.SI` 后缀（如 801010.SI），不带会查不到数据。
- 数据范围：2021-05-31 至今（跨 801010.SI/801030.SI/801080.SI 等多行业探测，最早均为 2021-05-31）
- 单次限量：无 3 天跨度限制，由 page/page_size 分页
- 提示：
  - `level` 必填（1/2/3），`start_date`/`end_date`/`industry_code` 必填。
  - 日期格式 **YYYYMMDD**；`industry_code` 必须带 `.SI` 后缀（如 801010.SI），不带会查不到数据。
  - 返回 `{code,message,data:{pageNum,pageSize,total,pages,records}}` 形态，字段 camelCase、全部为字符串。
  - 字段较多，下方输出参数表列核心列，完整字段见协议结构 `SwIndustryDailyMetricsItem`。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| level | int | Y | 行业层级：1/2/3 |
| start_date | string | Y | 起始日期，YYYYMMDD |
| end_date | string | Y | 截止日期，YYYYMMDD |
| industry_code | string | Y | 行业代码，带 .SI 后缀，如 801010.SI |
| page | int | N | 页码，从 1 开始 |
| page_size | int | N | 每页数量 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

## 调用方法（MCP）

> MCP 工具名 `ft_sw_industry_daily_metrics`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_sw_industry_daily_metrics","arguments":{"level": 1, "start_date": "20260101", "end_date": "20260717", "industry_code": "801010.SI"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_sw_industry_daily_metrics', {"level": 1, "start_date": "20260101", "end_date": "20260717", "industry_code": "801010.SI"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
