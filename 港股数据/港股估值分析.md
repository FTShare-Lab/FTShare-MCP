# 港股估值分析（MCP 工具 `ft_get_hk_valuatnanalyd`）

> **MCP 工具**：`ft_get_hk_valuatnanalyd`（category: `港股数据/行情数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：港股估值分析分页查询。可按 `trade_code` 查指定个股全部交易日的估值序列，或留空查全市场分页。每项含当日收盘价、成交量/额、波动率、港股股数/市值、换手率、PE/PS/PB/PCF/股息率等。提示：`trade_code` 可选：留空分页查全市场；指定则查该股全部交易日估值序列；与 F9 `market-cap-hk` 区别：本接口是按日估值时间序列（分页），F9 是单条最新一期快照。
- 数据范围：以服务端返回为准
- 单次限量：分页，默认 `page=1` / `page_size=20`，`page_size` 上限 500
- 提示：
  - 对外 v1 仅暴露 GET（底层 v0 同时支持 GET+POST）。
  - `trade_code` 可选：留空分页查全市场；指定则查该股全部交易日估值序列。
  - 与 F9 `market-cap-hk` 区别：本接口是按日估值**时间序列**（分页），F9 是单条最新一期快照。
  - `update_time` 为 int64 整数时间戳（格式如 `20251202214424`）。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| trade_code | string | N | 港股代码（支持 `700` / `00700.HK`）；留空查全市场 |
| page | int | N | 页码（从 1 开始），默认 1 |
| page_size | int | N | 每页数量，默认 20，上限 500 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |

| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_hk_valuatnanalyd`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_hk_valuatnanalyd","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_hk_valuatnanalyd', {})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
