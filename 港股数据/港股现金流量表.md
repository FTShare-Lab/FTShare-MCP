# 港股现金流量表（MCP 工具 `ft_hk_cashflow`）

> **MCP 工具**：`ft_hk_cashflow`（category: `港股数据/财务数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：港股现金流量表分页查询。所有过滤条件可自由组合：`stock_code` 单独 → 查该股全部报告期；`year` + `report_type` → 按年度+报告类型查全市场；`stock_code` + `year` + `report_type` → 精确查某股某年某报告期；`start_date`/`end_date` → 按报告期范围查。至少需要一个过滤条件，`year` 限制最近 3 年。提示：必须至少传一个过滤条件（`stock_code`、`start_date`、`end_date`、或 `year`+`report_type`）。
- 数据范围：以服务端返回为准
- 单次限量：分页，默认 `page=1` / `page_size=20`，`page_size` 上限 500；`year` 限制最近 3 年
- 提示：
  - 对外 v1 仅暴露 GET（底层 v0 同时支持 GET+POST）。
  - 必须至少传一个过滤条件（`stock_code`、`start_date`、`end_date`、或 `year`+`report_type`）。
  - 宽表结构：每个报告期一行，所有现金流科目横向展开；下表仅列元数据 + 各类代表字段，完整科目集以接口实际返回字段为准。
  - `report_type` 取值：`annual`（年报）/ `semi`（半年报）。
  - 内部字段（`id`/`company_id`/*_id/`op_mode`/`update_time` 等）不对外暴露。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| stock_code | string | N | 港股代码（如 `00700.HK`） |
| year | int32 | N | 报告期年份（如 2024），按 `end_date` 日历年过滤，需配 `report_type` |
| report_type | string | N | 报告类型：annual（年报）/ semi（半年报），需配 `year` |
| start_date | int32 | N | 报告期范围下界 YYYYMMDD（含），过滤 `end_date >= 此值` |
| end_date | int32 | N | 报告期范围上界 YYYYMMDD（含），过滤 `end_date <= 此值` |
| page | int | N | 页码，默认 1 |
| page_size | int | N | 每页数量，默认 20，上限 500 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

## 调用方法（MCP）

> MCP 工具名 `ft_hk_cashflow`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_hk_cashflow","arguments":{"stock_code": "00700.HK", "page": 1, "page_size": 3}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_hk_cashflow', {"stock_code": "00700.HK", "page": 1, "page_size": 3})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
