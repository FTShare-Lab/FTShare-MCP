# 美股利润表（MCP 工具 `ft_us_income`）

> **MCP 工具**：`ft_us_income`（category: `美股数据/财务数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询美股上市公司利润表数据。对外输出为长表 EAV 结构（每行一个财务科目，而非宽表）：`ind_name` 是科目名、`ind_value` 是科目值。数据按财年 + 报告期组织，跨股票按各自财年末可比。提示：`stock_code` 必填，纯代码不带交易所后缀（如 `NVDA`）；分页基于报告期数。
- 数据范围：2023-01 至今（参考美股现金流.md口径，跨 NVDA/AAPL 等多标的探测，最早均为 2023 年初）
- 单次限量：每页默认 50 条，最大 500 条；分页按**报告期**分页（page_size 限制报告期数，非科目行数）
- 提示：
  - `stock_code` 必填，纯代码不带交易所后缀（如 `NVDA`）。
  - 长表结构：`ind_name` 是科目名（底表英文字段名，如 revenue / net_income）、`ind_value` 是科目值，一个报告期展开为多行。
  - 分页基于报告期数，`items` 行数 ≈ 报告期数 × (60~90)，不能用 `total_items` 估算长表总行数。
  - 美股财年非自然年，`period` 匹配 `stmt_year`（财年字段），非日历年。
  - `report_type` 取值：`Q1` / `Q2` / `Q3` / `Q4` / `H1`（半年报）。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| stock_code | string | Y | 美股代码，纯代码不带后缀，如 NVDA |
| period | int | N | 财年（如 2024），匹配 stmt_year，非自然年 |
| report_type | string | N | 报告期类型：Q1 / Q2 / Q3 / Q4 / H1 |
| start_date | int | N | 报告期下界 YYYYMMDD（含），过滤 end_date |
| end_date | int | N | 报告期上界 YYYYMMDD（含），过滤 end_date |
| page | int | N | 页码，默认 1 |
| page_size | int | N | 每页报告期数，默认 50，最大 500 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

UsIncomeItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| stock_code | string | Y | 美股代码，如 NVDA |
| end_date | string | Y | 报告期截止日 YYYY-MM-DD |
| ind_type | string | Y | 报告期类型：Q1 / Q2 / Q3 / Q4 / H1 |

| ind_value | string | Y | 科目值（字符串，负值带 -） |
| report_type | string | Y | 报告类型：一季报 / 中报 / 三季报 / 年报 |

## 调用方法（MCP）

> MCP 工具名 `ft_us_income`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_us_income","arguments":{"stock_code": "AAPL", "page": 1, "page_size": 3}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_us_income', {"stock_code": "AAPL", "page": 1, "page_size": 3})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
