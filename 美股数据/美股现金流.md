# 美股现金流（MCP 工具 `ft_us_cashflow`）

> **MCP 工具**：`ft_us_cashflow`（category: `美股数据/财务数据`）。返回 **markdown 表格文本**（非结构化 JSON）。输入参数 / 输出参数 / 数据样例对照 [ftshare-doc](../../ftshare-doc/api-doc) 原接口。

- 描述：查询美股上市公司现金流量表数据，返回**长表 EAV** 结构（每行一个财务科目，而非宽表）。数据按**财年 + 报告期**组织，跨股票按各自财年末可比。典型返回：单股单报告期约 12-16 个非零科目行（经营活动净额、折旧摊销、资本支出、筹资活动净额等），数值字段为字符串、负值带 `-`（投资/筹资流出常见负值）。
- 数据范围：2023-01 至今（跨 NVDA/AAPL/KO/WMT/GE/MSFT/JPM/XOM/F/T 等多标的探测，最早均为 2023 年初）
- 单次限量：每页默认 50 条，最大 500 条；分页按**报告期**分页（page_size 限制报告期数，非科目行数）
- 提示：
  - `stock_code` 必填，纯代码不带交易所后缀（如 `NVDA`）。
  - 长表结构：`ind_name` 是科目名、`ind_value` 是科目值，一个报告期展开为多行。
  - 分页基于报告期数，`items` 行数 ≈ 报告期数 × (10~16)，不能用 `total_items` 估算长表总行数。
  - 美股财年非自然年，`period` 匹配 `stmt_year`（财年字段），非日历年。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| stock_code | string | Y | 美股代码，纯代码不带后缀，如 NVDA |
| period | int | N | 财年（如 2024），匹配 stmt_year，非自然年 |
| report_type | string | N | 报告期类型：Q1 / Q2 / Q3 / Q4 / H1 |
| start_date | int | N | 报告期下界 YYYYMMDD（含），过滤 end_date |
| end_date | int | N | 报告期上界 YYYYMMDD（含），过滤 end_date |
| page | int | N | 页码，默认 1 |
| page_size | int | N | 每页报告期数，默认 50，最大 500 |

## 输出参数

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| items | array | Y | 当前页记录列表（长表 EAV，每行一个科目） |
| total_pages | int | Y | 总页数（按报告期分页） |
| total_items | int | Y | 总报告期数（去重，非科目行数） |

UsCashflowItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| stock_code | string | Y | 美股代码，如 NVDA |
| end_date | string | Y | 报告期截止日 YYYY-MM-DD |
| ind_type | string | Y | 报告期类型：Q1 / Q2 / Q3 / Q4 / H1 |
| name | string | Y | 股票名称（中文） |
| ind_name | string | Y | 财务科目名（底表英文字段名） |
| ind_value | string | Y | 科目值（字符串，负值带 -） |
| report_type | string | Y | 报告类型：一季报 / 中报 / 三季报 / 半年报 / 年报 |

## 调用方法（MCP）

> MCP 工具名 `ft_us_cashflow`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，再 `tools/call`**，后续请求带该 header。返回 **markdown 表格文本**。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '')

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_us_cashflow","arguments":{"stock_code": "AAPL", "page": 1, "page_size": 3}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_us_cashflow', {"stock_code": "AAPL", "page": 1, "page_size": 3})
            print(res.content[0].text)   # markdown 表格文本

asyncio.run(main())
```
