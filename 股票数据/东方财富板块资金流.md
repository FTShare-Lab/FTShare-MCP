# 东方财富板块资金流（MCP 工具 `ft_get_eastmoney_sector_flow`）

> **MCP 工具**：`ft_get_eastmoney_sector_flow`（category: `股票数据/资金流向数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：获取东方财富板块（行业/概念/地域）日资金流数据，含主力/超大单/大单/中单/小单四档净流入及净占比。数据来源：东方财富。支持按板块代码（BK0488 等）、板块类型（industry/concept/regional）、交易日或日期区间过滤。提示：`sector_code` 为东财板块代码，以 BK 开头（如 BK0488）；`trade_date` 与 `start_date`/`end_date` 互斥。
- 数据范围：资金流类，最新一期或当日快照（历史序列下限以服务端返回为准）
- 单次限量：分页返回，默认 `page=1`、`page_size=50`、最大 `page_size=200`
- 提示：
  - `sector_type` 取值：industry（行业）/ concept（概念）/ regional（地域）。
  - `sector_code` 为东财板块代码，以 BK 开头（如 BK0488）。
  - `trade_date` 与 `start_date`/`end_date` 互斥。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| sector_code | string | N | 板块代码，如 BK0488 |
| sector_type | string | N | 板块类型：industry / concept / regional |
| trade_date | string | N | 交易日 YYYYMMDD；与 start_date/end_date 互斥 |
| start_date | string | N | 区间起始日 YYYYMMDD；需与 end_date 同时提供 |
| end_date | string | N | 区间结束日 YYYYMMDD；需与 start_date 同时提供 |
| page | int | N | 页码，从 1 开始，默认 1 |
| page_size | int | N | 每页条数，默认 50，最大 200 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_eastmoney_sector_flow`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_eastmoney_sector_flow","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_eastmoney_sector_flow', {})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
