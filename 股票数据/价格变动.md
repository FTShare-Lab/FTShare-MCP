# 价格变动（MCP 工具 `ft_get_price_change`）

> **MCP 工具**：`ft_get_price_change`（category: `股票数据/行情数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：单票 N 日涨跌幅接口。给定 `stock_code`、基准日 `base_date`、交易日数量 `n` 与方向 `direction`，自动过滤非交易日后计算区间涨跌幅。`forward` = 以基准日为结束日向前取 N 个交易日；`backward` = 以基准日为起始日向后取 N 个交易日。提示：`stock_code` 为 6 位纯数字代码（兼容沪/深/创业板/科创板）；建议查询区间 `n` 不超过 365 个交易日。
- 数据范围：按交易日计算（依赖底层交易日历与复权日线，覆盖标的上市以来交易日）
- 单次限量：单次一个标的、一个区间
- 提示：
  - `stock_code` 为 6 位纯数字代码（兼容沪/深/创业板/科创板）。
  - `direction` 仅接受 `forward` / `backward`。
  - 起止价格已按复权类型处理，保留 2 位小数。
  - 建议查询区间 `n` 不超过 365 个交易日。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| stock_code | string | Y | 目标股票 6 位纯数字代码，兼容沪市/深市/创业板/科创板 |
| base_date | string | Y | 涨跌幅计算基准日，格式 YYYYMMDD |
| n | uint32 | Y | 需计算的有效交易日数量，建议不超过 365 |
| direction | string | Y | 计算方向：forward（基准日为结束日）/ backward（基准日为开始日） |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

PriceChangeResponse：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| stock_code | string | Y | 目标股票 6 位纯代码 |
| stock_name | string | Y | 股票中文简称 |
| base_date | string | Y | 基准日期（YYYY-MM-DD） |
| n | uint32 | Y | 计算的有效交易日数量 |
| direction | string | Y | 计算方向（forward/backward） |
| start_date | string | Y | 涨跌幅区间起始交易日（YYYY-MM-DD，已过滤非交易日） |
| end_date | string | Y | 涨跌幅区间结束交易日（YYYY-MM-DD，已过滤非交易日） |
| actual_trading_days | uint32 | Y | 实际参与计算的有效交易日数量 |
| start_price | decimal | Y | 起始交易日收盘价（复权后，保留 2 位小数） |
| end_price | decimal | Y | 结束交易日收盘价（复权后，保留 2 位小数） |
| price_change | decimal | Y | 价格变动额 = end_price - start_price，保留 2 位小数 |
| change_pct | decimal | Y | N 日涨跌幅百分比 = price_change/start_price×100，保留 2 位小数 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_price_change`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_price_change","arguments":{"stock_code": "600519", "base_date": "20260620", "n": 5, "direction": "backward"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_price_change', {"stock_code": "600519", "base_date": "20260620", "n": 5, "direction": "backward"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
