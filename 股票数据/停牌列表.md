# 停牌列表（MCP 工具 `ft_suspension_list`）

> **MCP 工具**：`ft_suspension_list`（category: `股票数据/基础数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：返回当日所有停牌股票列表，包含盘前停牌（status 8）和盘中临时停牌（status 9）、全天停牌（status 10）。每条含停牌类型、停牌时间、复牌时间（盘中临停才有时间字段）。支持 `trade_date` 参数查询历史日停牌记录，默认当天。提示：`trade_date` 默认当天，可传历史交易日 YYYYMMDD；`page_size` 上限 200。
- 数据范围：当日停牌快照；支持 `trade_date` 查询历史日
- 单次限量：分页，`page` 默认 1，`page_size` 默认 50、最大 200
- 提示：
  - `trade_date` 默认当天，可传历史交易日 YYYYMMDD。
  - `suspend_time` / `resume_time` 仅盘中临时停牌（intra-day）有值，盘前/全天停牌为 null。
  - 响应为 code/message/data 信封，data 为分页对象（camelCase：pageNum/pageSize/total/pages/records）。
  - `page_size` 上限 200。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| trade_date | string | N | 交易日，格式 YYYYMMDD，默认当天 |
| page | int | N | 页码，从 1 开始，默认 1 |
| page_size | int | N | 每页数量，默认 50，最大 200 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

SuspensionInfo：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| symbol | string | Y | 标的代码 |
| suspension_type | string | Y | 停牌类型：pre-market（盘前停牌）/ intra-day（盘中临停）/ full-day（全天停牌） |
| suspend_time | string | Y | 停牌时间 HH:MM:SS，仅盘中临停有 |
| resume_time | string | Y | 复牌时间 HH:MM:SS，仅盘中临停有 |

## 调用方法（MCP）

> MCP 工具名 `ft_suspension_list`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_suspension_list","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_suspension_list', {})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
