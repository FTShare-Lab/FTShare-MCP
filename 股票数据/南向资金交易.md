# 南向资金交易（MCP 工具 `ft_southbound`）

> **MCP 工具**：`ft_southbound`（category: `股票数据/资金流向数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询南向资金（港股通）交易日交易数据。按交易日返回当日南向资金合计买入额/卖出额/净买额/成交笔数，以及沪港通（SH_HK）/ 深港通（SZ_HK）分市场通道明细。`date` 必填（YYYYMMDD）。提示：`date` 必填，格式 YYYYMMDD；金额字段单位为元（币种见 `currency` 字段）。
- 数据范围：按交易日，以服务端返回为准
- 单次限量：单个交易日一条聚合数据
- 提示：
  - `date` 必填，格式 YYYYMMDD。
  - 金额字段单位为元（币种见 `currency` 字段）；净买额 = 买入额 - 卖出额。
  - 响应为 code/message/data 信封结构。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| date | string | Y | 交易日，格式 YYYYMMDD |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| date | string | Y | 交易日 |
| currency | string | Y | 币种 |
| total | object | Y | 南向资金合计（见下） |
| channels | object | Y | 分市场通道（见下） |

total ：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| buy_amount | string | Y | 合计买入额，单位元 |
| sell_amount | string | Y | 合计卖出额，单位元 |
| net_buy_amount | string | Y | 合计净买额，单位元，公式 = 买入额 - 卖出额 |
| trade_count | int | Y | 合计成交笔数 |

channels ：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| SH_HK | object | Y | 沪港通通道数据 |
| SZ_HK | object | Y | 深港通通道数据 |

通道数据 ：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| buy_amount | string | Y | 该通道买入额，单位元 |
| sell_amount | string | Y | 该通道卖出额，单位元 |
| net_buy_amount | string | Y | 该通道净买额，单位元，公式 = 买入额 - 卖出额 |
| trade_count | int | Y | 该通道成交笔数 |

## 调用方法（MCP）

> MCP 工具名 `ft_southbound`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_southbound","arguments":{"date": "20260624"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_southbound', {"date": "20260624"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
