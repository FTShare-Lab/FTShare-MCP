# 机构持股明细（MCP 工具 `ft_get_stock_institution_holdings_detail`）

> **MCP 工具**：`ft_get_stock_institution_holdings_detail`（category: `股票数据/基础数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询 A 股单票机构持仓详细信息。按股票代码 + 年份 + 报告期 + 机构类型查询，返回该标的在指定报告期被各机构持仓的明细：机构名称、持股数量、市值、净值比例、流通比例、总股本比例等。`stock_code`、`report_type`、`inst_type`、`year` 均必填。提示：缺少必填参数时返回错误。
- 数据范围：按报告期，以服务端返回为准
- 单次限量：默认 `page=1`、`page_size=20`
- 提示：
  - `report_type` 取值：q1 / q2 / q3 / annual / announcement。
  - `inst_type` 取值：all_inst（返回全部）/ fund / qfii / insurance / social_security / securities / trust / other，非 all_inst 时按机构类型过滤。
  - 缺少必填参数时返回错误。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| stock_code | string | Y | 股票代码 |
| year | int | Y | 年份 |
| report_type | ReportType | Y | 报告类型：q1 / q2 / q3 / annual / announcement |
| inst_type | InstitutionType | Y | 机构类型：all_inst / fund / qfii / insurance / social_security / securities / trust / other |
| page | int | N | 页码，默认 1 |
| page_size | int | N | 每页条数，默认 20 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

## 调用方法（MCP）

> MCP 工具名 `ft_get_stock_institution_holdings_detail`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_get_stock_institution_holdings_detail","arguments":{"stock_code": "600519.SH", "year": 2024, "report_type": "annual", "inst_type": "fund"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_get_stock_institution_holdings_detail', {"stock_code": "600519.SH", "year": 2024, "report_type": "annual", "inst_type": "fund"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
