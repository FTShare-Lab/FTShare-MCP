# 涨停池（MCP 工具 `ft_limit_up_pool`）

> **MCP 工具**：`ft_limit_up_pool`（category: `股票数据/打板专题数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询涨停股池。返回涨停相关字段：涨停价、进入涨停时间点数组、炸板时间数组、首次封板时间、炸板次数；传历史交易日时返回该交易日曾涨停的股票。已过滤非股票标的（可转债等）。提示：仅返回当日有过涨停（`limit_up_enter` 非空）的股票；`trade_date` 为可选参数，格式 `YYYYMMDD`；不传或传当日时查询实时数据。
- 数据范围：当日实时快照，无历史序列（历史涨跌停查询见「昨日涨停池」）
- 单次限量：无分页，全量返回当日涨停股池
- 提示：
  - 仅返回当日有过涨停（`limit_up_enter` 非空）的股票。
  - 返回为裸 JSON 数组，无 code/message 信封。
  - 历日涨停股池请用「昨日涨停池」接口。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| - | - | - | 无入参 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

LimitUpPoolItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| symbol | string | Y | 标的代码 |
| trade_date | string | Y | 交易日 YYYYMMDD |
| ready | bool | Y | 当日涨跌停价格是否已就绪 |
| status | string | Y | 当前状态：normal / limit_up / limit_down |
| limit_up_price | decimal | Y | 涨停价，单位元 |
| limit_up_enter | array[string] | Y | 进入涨停的时间点数组，格式 HH:mm:ss |
| limit_up_break | array[string] | Y | 炸板时间数组，格式 HH:mm:ss |
| first_limit_up_time | string | Y | 首次封涨停时间 HH:mm:ss，无封板记录时为 null |
| limit_up_break_count | int | Y | 炸板次数（涨停封板后被打开次数） |

## 调用方法（MCP）

> MCP 工具名 `ft_limit_up_pool`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_limit_up_pool","arguments":{"trade_date": "20260717"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_limit_up_pool', {"trade_date": "20260717"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
