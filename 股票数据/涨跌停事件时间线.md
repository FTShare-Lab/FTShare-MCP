# 涨跌停事件时间线（MCP 工具 `ft_limit_event_timeline_3s`）

> **MCP 工具**：`ft_limit_event_timeline_3s`（category: `股票数据/打板专题数据`）。返回 **markdown 表格文本**（非结构化 JSON）。输入参数 / 输出参数 / 数据样例对照 [ftshare-doc](../../ftshare-doc/api-doc) 原接口。

- 描述：查询个股当日涨跌停事件 3 秒级时间线。每 3 秒将全市场涨跌停事件快照写入 ClickHouse，本接口读取内存中维护的最新涨跌停事件状态（`LIMIT_EVENT_MAP`），返回每只股票当日的涨跌停时间线：涨停价/跌停价、进入/炸板（涨停）/进入/翘板（跌停）时间点数组、首次封板/最后封板时间、炸板/开板次数、跌停封单资金等。支持传 `symbol` 查单只，不传则返回全市场（仅股票，过滤可转债等）。返回裸数组（非信封）。
- 数据范围：3 秒级实时，仅当日，无历史序列
- 单次限量：无分页，全量返回当前内存快照（单只或全市场）
- 提示：
  - `symbol` 可选，传则查单只；不传返回全市场（已过滤非股票标的）。
  - `symbol` 找不到或非股票标的时返回错误。
  - 价格字段：有对应涨/跌停进入记录时返回真实价，否则为 0。
  - 返回为裸 JSON 数组，无 code/message 信封。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| symbol | string | N | 标的代码，如 000001.XSHE；不传返回全市场 |

## 输出参数

StockLimitEventTimeline：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| symbol | string | Y | 标的代码 |
| trade_date | string | Y | 交易日 YYYYMMDD |
| ready | bool | Y | 当日涨跌停价格是否已就绪 |
| status | string | Y | 当前状态：normal / limit_up / limit_down |
| limit_up_price | decimal | Y | 涨停价，单位元；无涨停记录时为 0 |
| limit_down_price | decimal | Y | 跌停价，单位元；无跌停记录时为 0 |
| limit_up_enter | array[string] | Y | 进入涨停的时间点数组，格式 HH:mm:ss |
| limit_up_break | array[string] | Y | 炸板时间数组，格式 HH:mm:ss |
| limit_down_enter | array[string] | Y | 进入跌停的时间点数组，格式 HH:mm:ss |
| limit_down_break | array[string] | Y | 翘板时间数组，格式 HH:mm:ss |
| first_limit_up_time | string | Y | 首次封涨停时间 HH:mm:ss，无封板记录时为 null |
| limit_up_break_count | int | Y | 炸板次数（涨停封板后被打开次数） |
| last_limit_down_time | string | Y | 最后封跌停时间 HH:mm:ss，无封板记录时为 null |
| limit_down_break_count | int | Y | 开板次数（跌停封板后被打开次数，即翘板次数） |
| limit_down_seal_value | decimal | Y | 跌停封单资金，单位元；跌停时累加卖一跌停价挂单额，其余为 0 |

## 调用方法（MCP）

> MCP 工具名 `ft_limit_event_timeline_3s`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，再 `tools/call`**，后续请求带该 header。返回 **markdown 表格文本**。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '')

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_limit_event_timeline_3s","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_limit_event_timeline_3s', {})
            print(res.content[0].text)   # markdown 表格文本

asyncio.run(main())
```
