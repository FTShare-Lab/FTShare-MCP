# 集合竞价成交量（MCP 工具 `ft_stock_intraday_auction_volume`）

> **MCP 工具**：`ft_stock_intraday_auction_volume`（category: `股票数据/特色数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询全市场连续竞价成交占比，按分钟返回整体及分交易所的成交量、成交额占比分时序列。提示：`trade_date` 格式 YYYYMMDD；不传或传入当天时返回实时数据。
- 数据范围：以服务端返回为准（按交易日分钟级更新）
- 单次限量：分页返回，默认 `page=1`、`page_size=50`，`page_size` 最大 200
- 提示：
  - `trade_date` 格式 YYYYMMDD；不传返回当日实时数据，传入当日会降级为实时。
  - 每条记录为一分钟快照；`ts_millis` 为该分钟收盘时间戳（毫秒）。
  - `volume_ratio` = 该分钟成交量 / 全天总成交量；`turnover_ratio` = 该分钟成交额 / 全天总成交额。
  - 响应为 PaginatedApiResponse 信封（code/message/data，data 含 page_num/page_size/total/pages/records）。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| trade_date | string | N | 交易日期 YYYYMMDD；不传返回当日实时数据 |
| page | int | N | 页码，从 1 开始，默认 1 |
| page_size | int | N | 每页数量，默认 50，最大 200 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

AuctionVolumeMarketItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| ts_millis | int | Y | 该分钟收盘时间戳，单位毫秒 |
| overall | object | Y | 全市场（沪+深+北）聚合维度 |
| xshg | object | Y | 上交所维度 |
| xshe | object | Y | 深交所维度 |
| bjse | object | Y | 北交所维度 |

AuctionVolumeInner：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| volume | int | Y | 该分钟成交量 |
| volume_ratio | f64 | Y | 成交量占比 |
| turnover | string | Y | 该分钟成交额，单位元 |
| turnover_ratio | f64 | Y | 成交额占比 |

## 调用方法（MCP）

> MCP 工具名 `ft_stock_intraday_auction_volume`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_stock_intraday_auction_volume","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_stock_intraday_auction_volume', {})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
