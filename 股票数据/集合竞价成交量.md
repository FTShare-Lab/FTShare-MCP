# 集合竞价成交量（MCP 工具 `ft_stock_intraday_auction_volume`）

> **MCP 工具**：`ft_stock_intraday_auction_volume`（category: `股票数据/特色数据`）。返回 **markdown 表格文本**（非结构化 JSON）。输入参数 / 输出参数 / 数据样例对照 [ftshare-doc](../../ftshare-doc/api-doc) 原接口。

- 描述：查询连续竞价成交占比（全市场聚合模式），按分钟返回当日（或指定交易日）全市场及分交易所的成交量/成交额占比分时序列。不传 `trade_date` 时返回当日实时市场连续竞价分时占比；传入 `trade_date=YYYYMMDD` 时优先读取 `basedata.history_continuous_auction_volume_market` 历史表，未命中则按 `history_minute_str` / `history_day_str` 回补；若 `trade_date` 为今日则自动降级为实时数据（历史表尚未落库）。响应为 `PaginatedApiResponse<AuctionVolumeMarketItem>`（code/message/data 信封 + 分页），每条为一分钟的聚合快照，含整体（overall，沪+深+北）及上交所（xshg）/ 深交所（xshe）/ 北交所（bjse）分市场维度。
- 数据范围：以服务端返回为准（按交易日分钟级更新）
- 单次限量：分页返回，默认 `page=1`、`page_size=50`，`page_size` 最大 200
- 提示：
  - `trade_date` 格式 YYYYMMDD；不传返回当日实时数据，传入当日会降级为实时。
  - 每条记录为一分钟快照；`ts_millis` 为该分钟收盘时间戳（毫秒）。
  - `volume_ratio` = 该分钟成交量 / 全天总成交量；`turnover_ratio` = 该分钟成交额 / 全天总成交额。
  - 响应为 PaginatedApiResponse 信封（code/message/data，data 含 page_num/page_size/total/pages/records）。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| trade_date | string | N | 交易日期 YYYYMMDD；不传返回当日实时数据 |
| page | int | N | 页码，从 1 开始，默认 1 |
| page_size | int | N | 每页数量，默认 50，最大 200 |

## 输出参数

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| code | int | Y | 状态码：0 成功 |
| message | string | Y | 状态信息 |
| data | object | N | 分页对象（见下） |

data ：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| page_num | int | Y | 当前页码 |
| page_size | int | Y | 每页条数 |
| total | int | Y | 总记录数 |
| pages | int | Y | 总页数 |
| records | array[AuctionVolumeMarketItem] | Y | 分钟级市场聚合列表 |

AuctionVolumeMarketItem：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| ts_millis | int | Y | 该分钟收盘时间戳，单位毫秒 |
| overall | object | Y | 全市场（沪+深+北）聚合维度 |
| xshg | object | Y | 上交所维度 |
| xshe | object | Y | 深交所维度 |
| bjse | object | Y | 北交所维度 |

AuctionVolumeInner：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| volume | int | Y | 该分钟成交量 |
| volume_ratio | f64 | Y | 成交量占比 |
| turnover | string | Y | 该分钟成交额，单位元 |
| turnover_ratio | f64 | Y | 成交额占比 |

## 调用方法（MCP）

> MCP 工具名 `ft_stock_intraday_auction_volume`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，再 `tools/call`**，后续请求带该 header。返回 **markdown 表格文本**。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '')

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_stock_intraday_auction_volume","arguments":{}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_stock_intraday_auction_volume', {})
            print(res.content[0].text)   # markdown 表格文本

asyncio.run(main())
```
