# 龙虎榜明细（MCP 工具 `ft_abnormal_trading_details`）

> **MCP 工具**：`ft_abnormal_trading_details`（category: `股票数据/打板专题数据`）。返回统一 MCP 输出：`structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是同值的序列化 JSON，不额外返回 Markdown。输入参数 / 输出参数 / 数据样例见下文。
> 文中 `Response`、`items`、`records`、`code`、`message` 等名称仅为字段说明；MCP 对外固定为上述 `metadata/data`。

- 描述：查询龙虎榜明细（异动成交详情），按交易日返回当日上榜标的明细：含标的涨跌幅、收盘价、成交额，以及买入额/卖出额前 5 名营业部明细。传入 `date` 时返回指定交易日的数据。一次返回全部上榜标的），并补充标的名称。提示：`date` 格式 YYYYMMDD；金额字段单位为元。
- 数据范围：以服务端返回为准（按交易日更新）
- 单次限量：无入参分页，一次返回当日全部上榜标的
- 提示：
  - `date` 格式 YYYYMMDD；不传返回最新内存快照。
  - `top_buyers` / `top_sellers` 各为前 5 名营业部列表，含买入额/卖出额/净额。
  - 金额字段单位为元。

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
|------|------|------|------|
| date | string | N | 查询日期 YYYYMMDD；不传则使用当前内存快照 |

## 输出参数

> MCP 固定输出信封为 `structuredContent.metadata` + `structuredContent.data`；`content[0].text` 是与其同值的序列化 JSON，不是 Markdown。
>
> `items` / `records` / `code` / `message` 等传输字段不会直接出现在 MCP 结果中；分页与截断信息统一归入 `metadata`。

| MCP 字段 | 类型 | 必填 | 描述 |
|----------|------|------|------|
| metadata | object | Y | 契约版本、数据来源、工具名、业务口径、总量、分页、返回条数、截断状态及 warnings |
| data | array | Y | 归一化后的业务数据项；元素字段见下方 |

### data 业务字段

AbnormalTradingDetailResponse：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| symbol | string | Y | 标的代码，对外格式 symbol.suffix |
| symbol_name | string | N | 标的名称 |
| change_rate | f64 | Y | 涨跌幅，公式 = 涨跌额 / 前收盘价 |
| close | string | Y | 收盘价，单位元 |
| turnover | string | Y | 成交额，单位元 |
| top_buyers | array[Trader] | Y | 买入额前 5 名营业部 |
| top_sellers | array[Trader] | Y | 卖出额前 5 名营业部 |

Trader：

| 名称 | 类型 | 默认显示 | 描述 |
|------|------|---------|------|
| name | string | Y | 营业部名称 |
| buy | string | Y | 买入额，单位元 |
| sell | string | Y | 卖出额，单位元 |
| net | string | Y | 净额，单位元，公式 = 买入额 - 卖出额 |

## 调用方法（MCP）

> MCP 工具名 `ft_abnormal_trading_details`。MCP Streamable HTTP 要求**先 initialize 拿 `Mcp-Session-Id`，发送 `notifications/initialized`，再 `tools/call`**，后续请求同时带该 Session ID 和协商后的 `MCP-Protocol-Version`。返回统一 `metadata/data` 结构化输出；`content[0].text` 为同值 JSON 文本，不额外返回 Markdown。

**curl**：

```bash
SID=$(curl -sS -m 10 -D - -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-11-25","capabilities":{},"clientInfo":{"name":"t","version":"1"}}}' \
  | grep -i 'mcp-session-id:' | awk '{print $2}' | tr -d '\r')

curl -fsS -m 10 -o /dev/null -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

curl -sS -m 10 -X POST <MCP_BASE_URL> \
  -H "Accept: application/json, text/event-stream" -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SID" \
  -H "MCP-Protocol-Version: 2025-11-25" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"ft_abnormal_trading_details","arguments":{"date": "20260624"}}}'
```

**Python（`mcp` SDK，自动握手管理 session）**：

```python
import asyncio
from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async def main():
    async with streamablehttp_client('<MCP_BASE_URL>') as (r, w, _):
        async with ClientSession(r, w) as s:
            await s.initialize()
            res = await s.call_tool('ft_abnormal_trading_details', {"date": "20260624"})
            print(res.structuredContent)   # 推荐：统一 metadata/data 结构化输出
            print(res.content[0].text)   # 兼容：与 structuredContent 同值的 JSON 文本
asyncio.run(main())
```
